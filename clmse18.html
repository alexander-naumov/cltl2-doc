4.1 Type Speciﬁer Symbols Символы как спецификаторы типов

The type symbols deﬁned by the system include those shown in table 4.1. In addition, when a structure type is deﬁned using defstruct, the name of the structure type becomes a valid type symbol.

Символы определяющие типа определены системой, включая те, что перечислены в таблице 4.1. В дополнение, когда определяется структура с использованием defstruct, имя структуры автоматически становиться корректным символом типа.

Notice of correction. In the ﬁrst edition, the type speciﬁers signed-byte and unsigned-byte were inadvertently omitted from table 4.1.

В первой редактции спецификаторы типов signed-byte и unsigned-byte были неумышленно забыты в таблице 4.1.

X3J13 voted in March 1989 ⟨?⟩ to eliminate the type common; this fact is indicated by the brackets around the common type speciﬁer in the table.

X3J13 voted in March 1989 ⟨?⟩ to eliminate the type string-char; this fact is indicated by the brackets around the string-char type speciﬁer in the table.

X3J13 voted in March 1989 ⟨?⟩ to add the type extended-character and the type base-character.

X3J13 voted in March 1989 ⟨?⟩ to add the type speciﬁer real.

X3J13 votes have also implicitly added many other type speciﬁers as names of classes (see chapter 28) or of conditions (see chapter 29).
