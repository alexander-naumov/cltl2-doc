6.2 Предикаты типов данных

Возможно наиболее важными предикатами в Lisp’е эти предикаты, которые различают типы данных. То есть позволяют узнать принадлежит ли заданный объект данному типу, или могут сравнивать два спецификатора типов.

6.2.1 Общие предикаты типов

Если тип данных рассматривать, как множество все объектов, принадлежащих этому типу, тогда функция typep проверяет принадлежность множеству, тогда как subtypep — принадлежность подмножеству.

[Функция] typep  object type

typep является предикатом, которые истинен, если объект object принадлежит типу type, и ложен в противном случае. Следует отметить что объект может принадлежать нескольким типам, так как один тип может включать другой. type может быть любым спецификатором типа, описанным в главе 4, за исключением того, что он не может быть или включать список спецификатор типа, у которого первый элемент равен function или values. Спецификатор формы (satisﬁes fn) обрабатывается просто как применение функции fn к объекту object (смотрите funcall). Объект object принадлежит заданному типу, если результат не равен nil.

[Функция] subtypep  type1 type2

Аргументы должны быть спецификаторами типов, но только те которые могут использоваться и для typep. Два спецификатора типа сравниваются. Данный предикат истинен, если тип type1 точно является подтипом типа type2, иначе предикат ложен. Если результат nil, тогда тип type1 может быть, а может и не быть подтипом типа type2 (иногда это невозможно определить, особенно когда используется тип satisﬁes). Второе возвращаемое значение указывает на точность результата. Если оно является истиной, значит первое значение указывает на точную принадлежность типов. Таким образом возможны следующие комбинации результатов:

t t type1 точно является подтипом type2

nil t type1 точно не является подтипом type2

nil nil subtypep не может определить отношение

6.2.2 Специальные предикаты типов

Следующие предикаты осуществляют проверку определённых типов данных.

[Функция] null  object

null является истиной, если аргумент является (), иначе является ложью. Похожая операция производится not, однако not используется для отрицания булевых значение, тогда как null используется для проверки того, пустой ли список. Таким образом программист может выразить цели выбрав нужное имя функции.

(null x) ≡ (typep x ’null) ≡ (eq x ’())

[Функция] symbolp  object

symbolp является истиной, если её аргумент является символом, в противном случае является ложью.

(symbolp x) ≡ (typep x ’symbol)

[Функция] atom  object

Предикат atom является истиной, если аргумент не является cons-ячейкой, в противном случае является ложью. Следует отметить (atom ’()) являет истиной, потому что () ≡nil.

(atom x) ≡ (typep x ’atom) ≡ (not (typep x ’cons))

[Функция] consp  object

Предикат consp является истиной, если его аргумент является cons-ячейкой, в противном случае является ложью. Следует отметить, пустой список не является cons-ячейкой, так (consp ’()) ≡ (consp ’nil) ⇒ nil.

(consp x) ≡ (typep x ’cons) ≡ (not (typep x ’atom))

[Функция] listp  object

listp является истиной, если его аргумент является cons-ячейкой или пустым список (), в противном случае является ложью. Она не проверяет является ли «список Ъ (true list)» (завершающийся nil) или «dotted» (завершающийся не-null атомом).

(listp x) ≡ (typep x ’list) ≡ (typep x ’(or cons null))

[Функция] numberp  object

numberp является истиной, если аргумент любой вид числа, в противном случае является ложью.

(numberp x) ≡ (typep x ’number)

[Функция] integerp  object

integerp является истиной, если аргумент целое число, в противном случае является ложью.

(integerp x) ≡ (typep x ’integer)

[Функция] rationalp  object

rationalp является истиной, если аргумент рациональное число (дробь или целое), в противном случае является ложью.

(rationalp x) ≡ (typep x ’rational)

[Функция] ﬂoatp  object

ﬂoatp является истиной, если аргумент число с плавающей точкой, в противном случае является ложью.

(ﬂoatp x) ≡ (typep x ’ﬂoat)

[Функция] realp  object

X3J13 проголосовал в марте 1989 добавить функцию realp. realp истинна, если аргумент является действительным числом, иначе ложна.

(realp x) ≡ (typep x ’real)

[Функция] complexp  object

complexp является истиной, если аргумент комплексное число, в противном случае является ложью.

(complexp x) ≡ (typep x ’complex)

[Функция] characterp  object

characterp является истиной, если аргумент строковый символ, иначе является ложью.

(characterp x) ≡ (typep x ’character)

[Функция] stringp  object

stringp является истиной, если аргумент строка, иначе является ложью.

(stringp x) ≡ (typep x ’string)

[Функция] bit-vector-p  object

bit-vector-p является истиной, если аргумент битовый вектор, иначе является ложью.

(bit-vector-p x) ≡ (typep x ’bit-vector)

[Функция] vectorp  object

vectorp является истиной, если аргумент вектор, иначе является ложью.

(vectorp x) ≡ (typep x ’vector)

[Функция] simple-vector-p  object

vectorp является истиной, если аргумент простой общий вектор, иначе является ложью.

(simple-vector-p x) ≡ (typep x ’simple-vector)

[Функция] simple-string-p  object

simple-string-p является истиной, если аргумент простая строка, иначе является ложью.

(simple-string-p x) ≡ (typep x ’simple-string)

[Функция] simple-bit-vector-p  object

simple-bit-vector-p является истиной, если аргумент простой битовый вектор, иначе является ложью.

(simple-bit-vector-p x) ≡ (typep x ’simple-bit-vector)

[Функция] arrayp  object

arrayp является истиной, если аргумент массив, иначе является ложью.

(arrayp x) ≡ (typep x ’array)

[Функция] packagep  object

packagep является истиной, если аргумент является пакетом, иначе является ложью.

(packagep x) ≡ (typep x ’package)

[Функция] functionp  object

X3J13 проголосовал в июне 1988 определить
(functionp x) ≡ (typep x ’function)

Так как голосование также определило, что типы cons и symbol непересекаются с типом function, это было несовместимым изменением. Теперь functionp является ложной для символов и списков.

[Функция] compiled-function-p  object

compiled-function-p является истиной, если аргумент скомпилированный объект кода, иначе является ложью.

(compiled-function-p x) ≡ (typep x ’compiled-function)
